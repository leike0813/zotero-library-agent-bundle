# Synthesis Research Context

Use Synthesis commands for derived research structures, not as live Zotero mutation evidence.

## Choose the model

- Topics: list topics, find memberships by paper ref, then read context/report/review input for one topic.
- Graph overview: paged global summary. Slice: coherent bounded neighborhood. Layout: persisted coordinates. Metrics: metric rows for selected papers. Query-cluster: topic-scoped cluster. Ranking commands: external references versus library papers. Refresh-metrics: approval-gated repair, not a read.
- Library/reference indexes: paged derived index records. Resolver: turn tag, collection, or paper-ref selectors into a bounded paper set. Attention queue: ranked aggregate review candidates.
- Artifact manifest: discover paper-owned files. Artifact read: selected contents. Export-filtered: file delivery. Resolve-topic-digest: choose the digest associated with a topic paper.
- Concept and schema commands expose typed semantic models; they do not search raw Zotero metadata.

## Evidence and freshness

Record topic id, paper refs, graph/index cursor completion, resolver selectors and combine mode, artifact name/checksum, and model/schema identifier. Before reporting a current selection, attachment, Product, permission, or writeback fact, confirm it through the corresponding live Host Bridge command.

## Recovery

When a derived view looks stale, check `synthesis cache status` and `synthesis index status`. Do not invalidate cache or refresh graph metrics merely because a query returned no matches. Propose maintenance only when diagnostics identify the affected scope and the user can review the effect.
